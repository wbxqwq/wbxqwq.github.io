LLM推理

发表于2026-01-27|更新于2026-03-12
|字数总计:3.2k|阅读时长:10分钟|阅读量:

阅读本文之前，请先对Transformer、注意力机制具有一定的了解。可参考这篇文章：Attention进阶史（MHA, MQA, GQA, MLA） – 图神经网络公社

苏神的这篇文章：缓存与效果的极限拉扯：从MHA、MQA、GQA到MLA - 科学空间|Scientific Spaces

1.什么是VLM

传统的LLM只能处理纯文本类型的问题，但是不能读取图片中的信息。VLM可以理解为是LLM的Plus版本，它可以根据你提供给它的图片和问题综合来回答你。比如：你可以给他一张同时具有猫和狗的图片，问他请问这张图片中猫和狗在干什么？VLM可以根据这写信息来对你的提问进行回答。VLM是迈向MLLM的重要一步。

2. VLM的总体架构

当前主流的VLM模型（LLaVA、Qwen等）都是使一种“三明治”结构，即

​ VLM = Encodervis + Adapter + LLM

其中Vison Encoder作用是将原始图像 Ximg 转换为高维特征向量 Hvis；Adapter将视觉特征 Hvis 映射到 LLM 的文本嵌入空间 (Text Embedding Space)。让 LLM 把图片特征看作是一种“外语”token。LLM接收经过适配器（Adapter）转换的“视觉 Token”和用户的“文本 Token”，进行自回归 (Autoregressive) 推理。

2.1 视觉编码器（Vison Encoder）

传统的LLM将文本转换为文本Token，之后通过Transformer中的Decoder-only部分进行自回归推理。VLM实现的首要第一个关键就是如何让模型读取图片的信息，也就是把图片转换为包含信息的Token。当前主流的技术有两个：

  • ViT (Vision Transformer): 最常用的架构。将图片切割成 16 × 16 或 14 × 14 的 Patch，通过 Transformer Encoder 提取特征。论文可见：https://arxiv.org/pdf/2010.11929/100

ViT的思想非常简洁：对于一张图片，我们将他切割成很多个patch，对这些Patch，先通过一个线性投影，然后添加位置编码（Position Embeddings），最后将得到的Embedded Patches通过Transformer Encoder。

  • CLIP (Contrastive Language-Image Pre-training)：论文可见：https://proceedings.mlr.press/v139/radford21a/radford21a.pdf。关于CLIP更多算法细节后面可能会专门写一篇博客来讲（不一定），这里只是进行一个简单介绍。

    CLIP本身和ViT/ResNet有一个很大的区别，后两个本质是学习“图像到固定的 ID 映射”。这导致了两个问题：

    1. 高昂的标注成本: 需要人工标注大量数据 (Gold Labels)。
    2. 泛化能力弱: 只能识别训练集定义好的那 1000 个类别，遇到新类别必须重新训练最后一层。

    CLIP 摒弃了这种做法，转而利用互联网上随处可见的图像-文本对 (Image-Text Pairs)。它不让模型预测“类别 ID”，而是让模型学习：“哪段文本描述与这张图片最匹配？”

    对于一个包含N个图片-文本的数据对，CLIP是这样处理的：最初始的CLIP有两个Encoder，之后得到的特征拼在一起两两直接分别计算余弦相似度。不难发现我们的目标是最大化对角线的N个元素。基于此我们只需要训练Encoder即可。Text Encoder一般采用transformmer的Encoder结构，Image Encoder采取ViT或者ResNet。

    CLIP经过训练后，我们拿出其中的Image Encoder作为VLM的Vision Encoder，就是现在的技术。目前主流使用Vit-CLIP架构。

2.2 中间桥阶层（Adapter）/投影层 (Projection Layer)

这一部是连接底层视觉编码器 (Vision Encoder) 和顶层大型语言模型 (LLM) 的核心枢纽。它的根本任务是实现语言和视觉的模态对其。

当前使用的方法有三种：

适配器类型 (Adapter Type) 序列是否压缩？ 空间保留能力 训练难度 典型应用 (Typical Application)
MLP / Linear 否 (1:1 映射) 极强 (保留二维拓扑) 极低 需要细粒度理解的任务 (如 OCR, 文档问答)
Q-Former 是 (压缩至固定的 K 个) 较弱 较高 资源受限的通用图文对话、图像描述 (Captioning)

  • Linear / MLP Projection：

    • 代表模型：LLaVA 系列

    • 核心思想：这是目前最流行、也是结构最简单的方案。它直接将图像编码器输出的展平图块 (Flattened Patches) 视为序列，通过简单的神经网络层进行维度转换。

  • Q-Former / Querying Transformer

    • 代表模型：BLIP-2

    • 核心思想：引入一个轻量级的Querying Transformer（Q-Former），在冻结的图像编码器和语言模型之间架起桥梁。Q-Former 被设计为一个信息瓶颈，负责从图像编码器中提取对文本生成最有用的视觉特征，并将其传递给大型语言模型。

2.3 LLM

这个没什么特别好说的，就是一些Decoder-Only架构的经典LLM即可，一般就是经过pretrain过的一些基座模型：比如LLaMA、GPT-3等等。

训练流程

整体训练非为两部，当然这两步的前提是有已经训练好的LLM和Vision Encoder。之后VLM的训练主要集中在Adapter和微调上。训练流程简单概括为以下两个阶段：

  1. 先学词汇 (Stage 1): 冻结两端，只训练中间 Adapter，用海量低质量图文对，让模型学会“看图识物”。
  2. 后学逻辑 (Stage 2): 冻结视觉端，训练 Adapter 和 LLM (利用 LoRA)，用少量高质量对话数据，让模型学会“看图推理”。

需要注意的是，大模型的训练一般是及其困难的，我们一般只能对参数量较小的模型进行训练。大部分时候可能更加侧重于微调。所以大模型的训练笔者碍于知识有限，这里不做过多阐述，唯恐多说多错。

推理流程

在训练完成之后，VLM采用一套自回归的逻辑推理流程来进行生成。核心步骤和LLM类似但又有所不同。主要包含以下步骤：

1. 视觉特征提取 (Visual Feature Extraction)

当用户传入一张图像 Ximg 时，该图像首先经过预处理（如 Resize、Normalize），然后输入到底层的视觉编码器 (Vision Encoder)（例如 CLIP-ViT）。

  • 处理逻辑: ViT 将图像切分为 N 个 Patch，提取其特征。
  • 输出: 得到一个视觉特征矩阵 Zv ∈ ℝN × Dvis，其中 Dvis 是视觉模型的隐藏层维度（例如 CLIP-ViT-L 的 Dvis = 1024）。
  • 复杂度 (Complexity): 这一步涉及纯视觉的 Transformer 前向传播，由于自注意力机制，时间复杂度约为 O(N2 ⋅ Dvis)。由于在推理时这部分权重是固定的，通常只需计算一次并缓存。

2. 模态桥接与适配 (Modality Bridging via Adapter)

LLM 无法直接理解 Zv，因此需要通过适配器 (Adapter / Projection Layer) 将其映射到语言空间。

  • 处理逻辑: 将 Zv 通过训练好的线性层、MLP 或 Q-Former 进行维度转换和语义对齐。
  • 输出: 得到视觉词元 (Visual Tokens) Zv′ ∈ ℝN′ × DLLM。其中 DLLM 是顶层语言模型的词嵌入维度（例如 LLaMA-2 7B 的 DLLM = 4096）。
  • 注意: 如果使用简单的 MLP，N′ = N；如果使用 Q-Former 等压缩技术，N′ 会远小于 N。

3. 文本词元化 (Text Tokenization)

同时，用户输入的文本指令（Prompt，例如 “Please describe this image.”）会经过 LLM 自带的分词器 (Tokenizer)。

  • 处理逻辑: 将字符串切分为 Token ID，然后通过词嵌入矩阵 (Embedding Matrix) 转换为向量。
  • 输出: 得到文本词元 (Text Tokens) Etext ∈ ℝL × DLLM，其中 L 是文本的序列长度。

4. 上下文拼接 (Context Concatenation / Prefill Stage)

这是 VLM 推理中最关键的一步，称为预填充阶段 (Prefill Stage)。模型将视觉词元和文本词元在序列维度上直接拼接，形成统一的多模态上下文。

  • 统一表示: 输入序列 Hinput = [Zv′; Etext]。
  • 系统层面: 这个长为 N′ + L 的序列会一次性送入 LLM，计算所有的键值缓存 (Key-Value Cache, KV-Cache)。这一步计算量较大，但只需执行一次。

5. 自回归生成 (Autoregressive Generation / Decode Stage)

一旦预填充完成，LLM 就进入了解码阶段 (Decode Stage)，开始逐字生成回答。

  • 生成逻辑: 1. 给定历史上下文，模型预测下一个词元的概率分布：P(yt ∣ Hinput, y < t)。
    1. 根据特定的解码策略 (Decoding Strategy)（如Greedy Search）从概率分布中采样出一个词元 yt。
    2. 将新生成的 yt 转化为向量，拼接到输入序列中，进入下一轮循环。
  • 终止条件: 当模型生成了特定的序列结束标记 (End-of-Sequence, [EOS]) 或达到了设定的最大生成长度 (Max New Tokens) 时，推理循环终止。

KV Cache在VLM中的应用

关于KV Cache仍然可以参考https://gnn.club/?p=2729。

VLM中一大特点是：视觉Token的占比很大，所以关于通过优化KV Cache的策略，主要可以分为四个方面：

A. 架构设计层：从源头减少缓存体积 (Architectural Compression)

此类技术在模型预训练 (Pre-training) 阶段就已决定，属于结构性优化。

  • 多查询注意力 (Multi-Query Attention, MQA): 所有 Query 头共享同一个 K 头和 V 头。将 KV-Cache 的大小直接缩小了 h 倍。
  • 分组查询注意力 (Grouped-Query Attention, GQA): MQA 的折中方案，被 LLaMA-2/3 广泛采用。将 Query 头分组，每组共享一个 K 头和 V 头。它在模型表达能力 (Performance) 和 KV-Cache 显存压缩率之间取得了完美的平衡。

B. 系统调度层：解决显存碎片化 (System-Level Paging)

  • 代表技术: PagedAttention (vLLM 框架的核心)
  • 核心原理: 传统的推理框架为请求预先分配连续的大块显存，导致严重的内部碎片 (Internal Fragmentation) 和外部碎片。PagedAttention 借鉴了操作系统的虚拟内存分页思想，将 KV-Cache 划分为固定大小的块 (Blocks)。

C. 算法稀疏层：丢弃不重要的缓存 (Algorithmic Eviction & Sparsity)

这类技术属于无需重新训练的即插即用 (Plug-and-Play) 优化，旨在保持无限长的生成能力，但强制限制 KV-Cache 的最大阈值。

  • StreamingLLM (Attention Sink): 论文发现，注意力机制存在“注意力池” (Attention Sink) 现象——模型会把大量无关的注意力分数分配给序列的前几个 Token（如 <s>）。因此，只需在显存中永久保留前 4 个 Token 的 KV，并采用滑动窗口 (Sliding Window) 仅保留最近生成的 Token，就能在极小的常数级空间复杂度 O(1) 下实现无限长度的推理。
  • H2O (Heavy-Hitter Oracle): 类似于传统的 LRU 缓存淘汰算法。它动态评估每个 Token 被其他 Token 关注的频率（即积累的注意力分数）。只保留那些高频被查询的 “Heavy-Hitter” Token 的 KV-Cache，丢弃低价值的 Token。

D. 数据压缩层：量化 (Quantization)

  • KV-Cache 量化: 将 FP16 (16位) 的 K 和 V 张量压缩为 INT8 (8位)、FP8，甚至是 INT4 (4位)。
  • 效果: 显存占用直接减半甚至减少四分之三。目前如 TensorRT-LLM 已经成熟支持了 KV 缓存量化技术。

其中目前做的比较多的倾向于中间的两个：PagedAttention和通过Cache Eviction

Algorithmic Eviction & Sparsity

目前笔者主要在考虑Cache Eviction方面，所以仅在这个地方对部分文章的简单总结：

通过设计算法来进行KV的驱逐或者稀疏化，核心思想是：如何在不损失模型能力的前提下，通过算法层面的稀疏化 (Sparsity) 和驱逐 (Eviction) 极限压缩 KV Cache

这里我们同样需要，当前工作仍然分为大概两个部分：针对Token的压缩从而减小KV Cache的压力；以及不同Token入手，而是单纯从KV Cache的策略入手进行优化。感觉就是两个方面：压缩(针对Token)和驱逐(针对KV Cache)
